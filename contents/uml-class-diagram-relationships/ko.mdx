## UML 클래스 다이어그램 관계 요약표

6가지 관계가 있는데, 강약 마름모 2개(포집), 강약 화살표 2개(연의), 강약 속 빈 화살표 2개(일실)로 이뤄져있습니다.

| 관계           | 핵심 개념         | 다이어그램         | 키워드                        | 간단한 예시                                                                                                     |
| :------------- | :---------------- | :----------------- | :---------------------------- | :-------------------------------------------------------------------------------------------------------------- |
| **포함(복합)** | 강한 마름모       | 마름모(속 찬)      | **강한 포함 관계**, 필수 부품 | `집`과 `방`(집은 방을 포함하고 있다. 집이 없어지면 방도 없어진다.)                                              |
| **집합**       | 약한 마름모       | 마름모(속 빈)      | **약한 포함 관계**, 선택 부품 | `학과`와 `학생`(학과는 학생을 포함하고 있다. 학과가 없어져도 학생은 없어지지 않는다.)                           |
| **연관**       | 강한 화살표       | 화살표(실선)       | 멤버 변수, 지속적 관계        | `축구팀`과 `공격수`(축구팀은 공격수를 **항상** 사용한다.)                                                       |
| **의존**       | 약한 화살표       | 화살표(점선)       | **파라미터** , 잠깐 사용      | `개발자`와 `책`(개발자는 책을 **잠깐** 사용한다.)                                                               |
| **일반화**     | 강한 속 빈 화살표 | 속 빈 화살표(실선) | 상속, 부모-자식               | `고양이`는 `동물`이다. (동물은 고양이보다 더 **일반적** 이다. 상위(동물), 하위(고양이)로 나뉨, 동물 \<- 고양이) |
| **실체화**     | 약한 속 빈 화살표 | 속 빈 화살표(점선) | 인터페이스, 약속 이행         | `로봇`은 `청소할 수 있다`.(청소할 수 있는 **'기능'** 으로 그룹화. '청소할 수 있는' \<-- '로봇')                 |

## 관계별 클래스 다이어그램

### 포함

- 키워드 : **생명주기 일치**, 필수 부품(포함=복합)
- 예시 : 집과 방(집이 없어지면 방도 없어진다.)

<MermaidDiagram
  chart={`classDiagram
    direction LR

    class House["집"]
    class Room["방"]
    House *--> Room : 포함

`}
/>

### 집합

- 키워드 : **생명주기 독립**, 선택 부품
- 예시 : 학과와 학생(학과가 없어져도 학생은 없어지지 않는다.)

<MermaidDiagram
  chart={`classDiagram
    direction LR

    class Department["학과"]
    class Student["학생"]
    Department o--> Student : 집합

`}
/>

### 연관

- 키워드 : 멤버 변수, 지속적 관계
- 예시 : 축구팀과 공격수(축구팀은 공격수를 항상 사용한다.)

<MermaidDiagram
  chart={`classDiagram
    direction LR

    class Student["학생"]
    class Professor["교수"]
    Student --> Professor : 연관

`}
/>

### 의존

- 키워드 : 메소드 파라미터, 잠깐 사용
- 예시 : 개발자와 책(개발자는 책을 잠깐 사용한다.)

<MermaidDiagram
  chart={`classDiagram
    direction LR

    class Developer["개발자"]
    class Book["책"]
    Developer ..> Book : 의존

`}
/>

### 일반화

- 키워드 : 상속, 부모-자식
- 예시 : 고양이는 동물이다. (동물은 고양이보다 더 **일반적** 이다. 상위(동물), 하위(고양이)로 나뉨, 동물 \<- 고양이)

<MermaidDiagram
  chart={`classDiagram
    direction LR

    class Animal["동물"]
    class Cat["고양이"]
    Animal <|-- Cat : 일반화

`}
/>

### 실체화

- 키워드 : 인터페이스, 약속 이행
- 예시 : 로봇은 청소할 수 있다. (청소할 수 있는 **'기능'** 으로 그룹화. '청소할 수 있는' \<-- '로봇')

<MermaidDiagram
  chart={`classDiagram
    direction LR

    class Flyable["청소할 수 있는"] {
      <<interface>>
    }
    class Robot["로봇"]
    Flyable <|.. Robot : 실체화

`}
/>
